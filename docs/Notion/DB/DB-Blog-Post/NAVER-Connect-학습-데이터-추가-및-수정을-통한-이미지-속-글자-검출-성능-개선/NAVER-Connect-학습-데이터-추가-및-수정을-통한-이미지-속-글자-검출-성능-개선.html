    • 테스트셋은 대회 참가자에게 별도로 노출되지 않음

개요

Info

https://stages.ai/competitions/224/overview/description

스마트폰으로 카드를 결제하거나, 카메라로 카드를 인식할 경우 자동으로 카드 번호가 입력되는 경우가 있습니다. 또 주차장에 들어가면 차량 번호가 자동으로 인식되는 경우도 흔히 있습니다. 이처럼 OCR (Optimal Character Recognition) 기술은 사람이 직접 쓰거나 이미지 속에 있는 문자를 얻은 다음 이를 컴퓨터가 인식할 수 있도록 하는 기술로, 컴퓨터 비전 분야에서 현재 널리 쓰이는 대표적인 기술 중 하나입니다.

(출처 : 위키피디아)

OCR task는 글자 검출 (text detection), 글자 인식 (text recognition), 정렬기 (Serializer) 등의 모듈로 이루어져 있습니다. 본 대회는 아래와 같은 특징과 제약 사항이 있습니다.

  • 본 대회에서는 ‘글자 검출’ task 만을 해결하게 됩니다.
  • 예측 csv 파일 제출 (Evaluation) 방식이 아닌 model checkpoint 와 inference.py 를 제출하여 채점하는 방식입니다. (Inference) 상세 제출 방법은 AI Stages 가이드 문서를 참고해 주세요!
  • 대회 기간과 task 난이도를 고려하여 코드 작성에 제약사항이 있습니다. 상세 내용은 베이스라인 코드 탭 하단의 설명을 참고해주세요**.**
  • Input : 글자가 포함된 전체 이미지
  • Output : bbox 좌표가 포함된 UFO Format (상세 제출 포맷은 평가 방법 탭 및 강의 5강 참조)

본론

EDA

  • ICDAR19(MLT, LSVT, ArT)

    • 곡선과 원형 패턴의 글자 이미지들이 많이 들어가 있음
    • Image size (2000, 2000) ~ (4000, 4000) 사이의 다양한 사이즈로 구성
    • ICDAR19 - MLT
      • 학습 시간을 고려하여 Korean only 최종 채택 (약 1,000장)
    • ICDAR19 - LSVT
      • 학습 시간을 고려하여 사용하지 않음
    • ICDAR19 - ArT
      • 캠퍼 제작 데이터셋과 가장 유사한 Annotation rule
      • 학습 시간을 고려하여 사용하지 않음

  • Camper Dataset

    • Annotation 실습으로 Camper들이 직접 제작한 Dataset
    • Annotation이 깔끔한 좋은 데이터가 있는 반면 어노테이션이 불안정한 데이터도 포함
    • Annotation 보정 없이 그대로 전체 사용
      • 일부 실험에서 제외하였을 경우도 평가함
      • 본 대회의 annotation 기준과 가장 유사하다고 판단

  • HierText

    • CVPR 2022 / Towards End-to-End Unified Scene Text Detection and Layout Analysis @Google Research 에서 발표
    • Scene Text Detection Task 와 Layout Analysis 을 통합하기 위한 모델과 관련한 논문과 데이터셋을 공개
    • 특징

      • 이미지당 평균 100단어 이상

      • 현재 가장 밀도가 높은 TextOCR(2021년)보다 두 배 더 밀도가 높음

Data

  • JPEG 이미지 메타정보(exif) Orientation 변환

  • 다각형 Annotation → Rect 형식으로 변환

Test

  • Augmentation

    • 다양한 실험을 기반으로 학습 시 최적의 Augmentation을 찾음
    • Geometric Augmentation
      • Rotate : ±30º
      • Random Crop : 20% ~ 100%
    • Channel Augmentation
      • Color Jitter
      • 성능 하락으로 인해 사용하지 않음
        • Gauss Noise, ISO, Channel Shuffle, CHALE, ImageCompression 등
    • Input Augmentation Visualization
      • Input 이미지를 시각화 함으로써 각 Augmentation 적용 시 어떤 효과가 있는지 눈으로 확인

  • Model Ensemble

    • Ensemble Method

      • NMS(None Maximum Suppression) 사용

      • 각 모델에서 나온 추론 결과를 모아 NMS를 진행하여 최종 Output 결정

      • Output Visualization

        • 모델 추론 결과를 시각화하여 모델이 어떤 것들을 잘 예측하고 어떤 문제가 있는지 파악

        • 이를 활용하여 다음 스텝을 계획하거나 앙상블에 필요한 모델의 장점을 파악

      • Multi Scale Inference

        • 입력 이미지 크기에 따라 성능이 달라짐
        • 장단점을 확인하고 입력 이미지의 크기를 다르게 하여 Multi Scale Inference 진행

      • Model Ensemble

        • 여러 실험 결과의 장점을 살리고자 앙상블 진행

    • Ensemble Flow Diagram

결론

  • 결과

    • public f1: 0.6901 → private f1: 0.7967, 1위

  • 느낀점

    • AI 모델 학습에서 데이터가 얼만큼 중요한지 경험을 통해 알게 되었다.
    • 강건한 모델은 좋은 데이터로부터 만들 수 있다고 느꼈다.
      • HierText dataset을 통해 일반화 성능을 높인것이 Public rank 6위에서 Private rank 1위를 할 수 있게 만들어 주었다고 판단
    • 많은 데이터보다도, 잘 구성된 웰메이든 데이터셋이 더 큰 성능을 보였다.
      • 다른 캠퍼들에 비해 적은 데이터를 사용하였지만, 더 높은 성능을 얻었다.
